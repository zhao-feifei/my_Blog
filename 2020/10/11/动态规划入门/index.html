关于动态规划的一点个人理解

一.什么是动态规划？

不做概念复读机，用自己的话说，动态规划就是将原问题拆分为子问题的一种方法。这种方法在计算机科学，数学，经济学等领域广泛应用。举个例子，1+1=？，答案是2，那么，1+1+1=？此时，我们不需要重新计算1+1的值，只需要在2的基础上加1即可。

二.动态规划的解题步骤

注：这里特指利用动态规划解算法题的步骤

1.搞清楚dp数组的含义

我们需要创建一个dp数组（dp是Dynamic programming的缩写）用来存储已知的历史记录，比如上面的1+1=2，计算1+1+1的时候我们就可以直接利用已经算出的历史记录，节省时间，那么首先要搞清楚dp数组中的元素代表的是什么含义？

2.写出状态转移方程

这是最难的一个步骤，一般情况下写出状态转移方程就已经成功了一半，其实就是dp数组中各元素之间的关系式，比如如上面例子的状态转移方程就是dp[i]=dp[i-1]+1

3.找到初始值

我们已经找到了元素之间的关系，只要给出一个初始值，就能逐个递推，最后算出结果dp[n]，而dp[n]也就是我们想求出的东西。

三.两道例题

例题1

小试牛刀：假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

给定 n 是一个正整数。

例：

输入：3

输出：3

解释：有三种方法可以爬到楼顶

  1. ​ 1 阶 + 1 阶 + 1 阶
  2. ​ 1 阶 + 2 阶
  3. ​ 2 阶 + 1 阶
1.dp[i]的含义

按照之前的三个步骤，我们最终要计算的是爬到n阶台阶一共有多少种方法，那么，

dp[i]表示爬上i阶台阶的方法数。

2.状态转移方程

假设我爬到了第10阶台阶，显然，只有两种可能，我是从第9阶上了一步到达第10阶，或者是从第8阶上了两步到达第10阶，所以， 爬上第10阶的方法数 = 爬上第9阶的方法数 + 爬上第8阶的方法数 ，很容易写出状态转移方程：

dp [ i ] = dp [ i - 1 ] + dp [ i - 2]

3.初始值

爬上第1阶有一种方法，第二阶有两种方法，分别给dp[1]和dp[2]赋值即可

例题2

渐入佳境：给定一个包含非负整数的 m x n网格 grid ，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

说明：每次只能向下或者向右移动一步。

示例：

输入：grid = [[1,3,1],[1,5,1],[4,2,1]]

输出：7

解释：因为路径 1→3→1→1→1 的总和最小。

1.dp[i] [j]的含义

这里用的是二维数组,dp[i] [j]表示走到当前位置的最小路径和。

2.状态转移方程

题目规定了只能向下或者向右走，那么当前位置只能由上边或者左边的格子走过来

因此状态转移方程：

dp[i][j] = Math.min(dp [ i - 1 ] [ j ], dp [ i ] [ j - 1 ]) + grid [ i ] [ j ]

注：grid [ i ] [ j ]表示当前格子中的数字

3.初始值

dp [0] [0]的值就是给定数组grid [0] [0]的值，需要注意要对第一行和第一列的dp数组进行初始化。
